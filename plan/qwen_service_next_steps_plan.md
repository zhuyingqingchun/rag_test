# Qwen 服务系统下一步计划

## 1. 当前结论

- Qwen 服务已经成功拉起，`/v1/models` 接口可正常返回。
- `test_openai_client.py` 在关闭代理变量后已经测试通过。
- 本轮已定位并修复若干关键问题，包括：
  - `run_server.py` 缺少导入
  - `health_check` URL 拼接重复 `/v1`
  - `wait_for_startup` 端口判断逻辑错误
  - 运行环境中 `vllm` 安装位置不一致
- 当前剩余的主要问题不是模型本身，而是**服务管理脚本的一致性和健壮性**。

## 2. 下一步目标

将当前“能跑通”的状态，整理成“可重复启动、可稳定检查、可交付”的 Python 核心服务系统。

## 3. 下一步任务清单

### 任务 1：统一 Python 解释器路径

目标：所有核心脚本统一使用同一个解释器。

执行项：
- 检查 `config.py` 中的 `python_bin`
- 确保 `run_server.py`
- `stop_server.py`
- `status_server.py`
- `test_openai_client.py`
  都只依赖该解释器
- 避免再次出现“系统 Python 没装 vllm，但 conda 环境装了”的问题

验收标准：
- 任意脚本均可直接通过同一 Python 路径执行
- 不再出现 `ModuleNotFoundError: vllm`

### 任务 2：修复 PID 文件生命周期

目标：让状态检查、停止服务、重启服务都依赖真实有效的 PID 文件。

执行项：
- 检查 `write_pid_file()` 写入时机
- 检查 `delete_pid_file()` 删除时机
- 明确父进程退出、子进程存活时 PID 应保存哪个进程
- 让 `status_server.py` 能基于 PID + HTTP 健康检查联合判断

验收标准：
- 服务启动后，PID 文件存在且内容正确
- `status_server.py` 能正确显示“运行中”
- `stop_server.py` 能通过 PID 正确停止服务

### 任务 3：标准化健康检查逻辑

目标：统一 base URL 与健康检查接口拼接规则。

执行项：
- 约定 `base_url` 是否包含 `/v1`
- `health_check()` 内部不再重复拼接 `/v1`
- 所有脚本对健康检查均统一调用同一工具函数

验收标准：
- 不再出现 `.../v1/v1/models`
- `status_server.py`、`test_openai_client.py`、`run_server.py` 的检查逻辑一致

### 任务 4：处理代理变量影响

目标：避免本地回环地址调用被代理污染。

执行项：
- 在 Python 请求代码中显式禁用本地代理影响，或
- 在脚本中对 `127.0.0.1` / `localhost` 请求禁用代理
- 将这一行为写入 README

验收标准：
- 在存在 `http_proxy/https_proxy/ALL_PROXY` 环境变量时，本地接口测试仍能通过
- 不再依赖手工 `unset` 才能测试成功

### 任务 5：补齐状态脚本稳定性

目标：让 `status_server.py` 成为可信状态入口。

执行项：
- 修正缺失导入问题
- 输出以下信息：
  - 服务是否运行
  - PID
  - 端口是否监听
  - `/v1/models` 是否可访问
  - 当前模型名
- 增加异常输出说明

验收标准：
- 服务启动时显示运行中
- 服务关闭时显示未运行
- 输出信息清晰、无歧义

### 任务 6：形成最小交付闭环

目标：让别人只看 README 就能完成部署、启动、测试。

执行项：
- 更新 README
- 增加“环境准备”章节
- 增加“首次启动”章节
- 增加“测试命令”章节
- 增加“常见问题”章节

README 至少应包含：
- 模型路径配置方法
- Python 环境要求
- vllm 安装方式
- 启动命令
- 停止命令
- 状态检查命令
- API 测试命令
- 代理问题说明

验收标准：
- 新人可按 README 独立完成一次完整启动和测试

## 4. 建议优先级

### P0（先做）
- 统一 Python 解释器路径
- 修复 PID 文件逻辑
- 统一健康检查逻辑

### P1（随后做）
- 处理代理变量影响
- 补齐 `status_server.py` 输出

### P2（最后做）
- README 完善
- 增加重启脚本或一键自检脚本

## 5. 建议新增文件

可新增一个：

- `self_check.py`

功能：一次性检查以下内容：
- Python 解释器路径
- vllm 是否可导入
- 模型目录是否存在
- 端口是否占用
- `/v1/models` 是否可访问
- 当前代理变量是否影响本地请求

这样可以把部署前排查时间进一步缩短。

## 6. 最终目标

把当前项目从“已能跑通的服务原型”，提升为：

**基于 Python 核心脚本的可启动、可停止、可检测、可测试、可复现的 Qwen 模型服务系统。**
