# 最小闭环下一步报告：文档/知识库输入 → 检索 → Qwen 结构化报告

## 1. 当前状态

目前 Qwen 服务侧已经完成工程化基础能力，包括：

- 服务可启动、可停止、可检测、可测试、可复现
- 健康检查、PID 生命周期、代理变量处理、自检脚本已经补齐
- `/v1/models` 接口和基础生成测试已经通过

这说明**模型服务底座已可用**，下一阶段应从“服务可用”切换到“任务可用”。

---

## 2. 下一步目标

构建一个**最小业务闭环**，验证整套流程是否真正能完成“检索增强报告生成”任务。

闭环定义：

1. 输入一份或多份文档，或一组知识库文本
2. 对文档进行切分与索引
3. 根据用户问题召回相关片段
4. 将召回片段和任务指令一起发送给 Qwen
5. 输出固定格式的结构化报告

---

## 3. 本阶段建议范围

为了尽快落地，本阶段先**不做复杂框架接入**，只做最小版本：

- 仅支持 `txt / md / pdf(可后补)` 三类输入
- 检索先用最简单的 `BM25` 或 `TF-IDF`
- 先不接向量库
- 先不做前端页面
- 先在命令行完成 end-to-end 验证
- 输出固定模板报告：标题、摘要、关键点、结论、引用片段

---

## 4. 最小闭环的模块拆分

### 模块1：文档导入
职责：
- 读取本地文档
- 支持单文件与目录读取
- 提取纯文本内容

建议输出：
- `docs_raw.jsonl`

### 模块2：文本切分
职责：
- 按段落或固定长度切分文本
- 为每个 chunk 分配 `doc_id / chunk_id / source`

建议输出：
- `docs_chunks.jsonl`

### 模块3：基础检索
职责：
- 根据用户 query 在 chunk 中检索 top-k
- 返回最相关的 3~5 个片段

建议输出：
- `retrieved_context.json`

### 模块4：报告生成
职责：
- 构造 prompt
- 调用 Qwen OpenAI-compatible 接口
- 返回固定结构报告

建议输出：
- `report.md`

### 模块5：结果记录
职责：
- 保存 query、召回片段、最终报告、时间戳
- 为后续评估和复现保留记录

建议输出：
- `runs/<timestamp>/`

## 6. 推荐先做的 5 个脚本

### 1) ingest.py
作用：
- 扫描输入目录
- 读取文本文件
- 输出统一格式 JSONL

### 2) chunk.py
作用：
- 将文档切成 chunk
- 保存 chunk 元数据

### 3) retrieve.py
作用：
- 输入 query
- 返回 top-k 相关 chunk

### 4) generate_report.py
作用：
- 读取召回片段
- 调用 Qwen
- 输出结构化报告

### 5) pipeline_demo.py
作用：
- 串起全流程
- 一条命令完成：导入 → 切分 → 检索 → 生成

---

## 7. 结构化报告模板（第一版）

建议固定成下面格式：

```markdown
# 报告标题

## 一、任务说明
- 用户问题：
- 生成时间：

## 二、检索摘要
- 共召回片段数：
- 主要来源：

## 三、关键内容整理
1. ...
2. ...
3. ...

## 四、综合分析
...

## 五、结论
...

## 六、参考片段
- [1] ...
- [2] ...
- [3] ...
```

这样做的好处是：
- 输出稳定
- 容易评估
- 后面容易扩展成 Word/PDF 导出

---

## 8. 本阶段验收标准

完成后，至少要满足下面 6 条：

1. 能读取指定目录中的文档
2. 能生成 chunk 数据文件
3. 能输入 query 并返回 top-k 相关片段
4. 能调用 Qwen 服务生成结构化报告
5. 能把报告保存为 markdown 文件
6. 能把本次运行输入、检索结果、报告全部留档

---

## 9. 本阶段不要做的事

为了防止一开始做太重，这一轮先不要做：

- 不做前端界面
- 不做向量数据库
- 不做复杂 rerank
- 不做多用户系统
- 不做权限管理
- 不做多格式导出优化
- 不做异步任务队列

先把“最小闭环跑通”比什么都重要。

---

## 10. 最自然的执行顺序

### 第一步
先准备 3~5 份测试文档，内容尽量短而且主题明确。

### 第二步
完成 `ingest.py + chunk.py`

### 第三步
完成一个最简单的 `retrieve.py`

### 第四步
完成 `generate_report.py`，让 Qwen 根据检索片段输出固定模板报告

### 第五步
写 `pipeline_demo.py`，一条命令走完整流程

---

## 11. 一句话结论

下一步不是继续修服务，而是基于现有 Qwen 服务，先做一个**纯 Python 的最小 RAG 报告闭环原型**。  
只要这一步跑通，后面再扩展知识库、向量检索、报告编排、导出和界面，都会顺很多。
