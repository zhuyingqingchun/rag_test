# RAG 系统简介

## 什么是 RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合了信息检索和语言生成的技术。它通过从外部知识库中检索相关信息，然后将这些信息作为上下文提供给语言模型，从而生成更准确、更有依据的回答。

## RAG 的核心组件

### 1. 检索器（Retriever）

检索器负责根据用户查询从外部知识库中检索出最相关的文本片段。常用的检索方法包括：

- **BM25**：基于统计的检索算法，考虑词频和逆文档频率
- **TF-IDF**：词频-逆文档频率加权方法
- **向量检索**：基于嵌入向量的语义相似度检索

### 2. 知识库（Knowledge Base）

知识库存储大量的结构化或非结构化文本数据，作为检索器的信息来源。它可以是：

- 文档集合（PDF、Word、TXT 等）
- 数据库记录
- 网页内容
- 企业内部文档

### 3. 生成器（Generator）

生成器通常是大型语言模型（LLM），它接收用户查询和检索到的相关文本片段，综合理解后生成准确、流畅且基于证据的回答。

## RAG 的工作流程

1. **文档导入**：将外部知识库文档导入系统
2. **文本切分**：将长文档切分成小的文本块（chunks）
3. **索引构建**：为文本块建立索引（可选向量索引）
4. **查询检索**：用户提问时，检索相关文本块
5. **生成回答**：将查询和检索结果一起发送给 LLM 生成回答

## RAG 的优势

- **减少幻觉**：基于实际检索到的信息生成回答
- **知识更新**：只需更新知识库即可，无需重新训练模型
- **可解释性**：可以追溯回答的来源
- **领域适应**：通过特定领域文档快速适应新领域

## RAG 的挑战

- **检索质量**：检索结果的质量直接影响生成效果
- **上下文长度**：LLM 的上下文长度限制了可使用的检索结果数量
- **延迟**：检索步骤会增加响应时间
- **成本**：需要额外的存储和计算资源

## 应用场景

- **问答系统**：企业知识库问答
- **文档摘要**：长文档自动摘要
- **内容创作**：基于参考资料的写作
- **客服助手**：产品文档和FAQ问答
